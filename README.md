## 1. アプリコンセプト

### 1.1 キャッチコピー

**「切り取って、貼って、私になる。」**

断片を切り取るという行為・スクラップブックに貼るという行為・積み重なって自分が見えてくるという結果、の3つを1行に凝縮したコピー。

---

### 1.2 コアコンセプト

**「日常の断片を切り取り、貼り重ねる、デジタルスクラップブック。」**

かつて雑誌の切り抜きをノートに貼っていたように、スマホで撮った日常の断片を切り取ってブックに貼っていく。気になったものを集め続けるうちに、それが記憶の蓄積になり、自分だけのコレクションになる。

---

### 1.3 世界観

**アナログの温かさ × デジタルの便利さ**

| アナログの要素 | デジタルの要素 |
|---|---|
| リングノートの質感 | スマホで即保存 |
| 切り抜きを貼る感覚 | 正方形トリミング |
| ページをめくる体験 | どこからでも見返せる |
| 手帳に育てていく | フレンドとゆるく共有 |

---

### 1.4 Scrappaが解決する課題

現在のユーザーは「気になったものを残したい」という欲求を、以下のような非効率な方法で代用している。

| 悩み | 現在の代用ツール | Scrappaの解決 |
|---|---|---|
| 撮った写真が埋もれる | カメラロール | ブックとして積み重なる・見返せる |
| 整理が続かない | Notion | 整理しなくていい・貼るだけ |
| 作り込みが大変 | GoodNotes | 正方形クロップだけ・シンプル |
| 発信したくないものが埋もれる | Instagram保存 | 自分のためだけのスペース |
| 自分で撮った写真が馴染まない | Pinterest | 自分の断片だけで構成できる |

## 2. 提供する体験

### 2.1 体験の流れ

```
① 出会い
  街で気になるものを見かける
  「なんかいいな」「これ好きかも」
  ↓
② 切り取り
  撮影して、気になる部分だけトリミング
  全体じゃなくていい、断片でいい
  ↓
③ 積み重なり
  ブックにクリップが増えていく
  意識せずに続けているうちにページが埋まっていく
  ↓
④ 発見
  ふと見返したとき、クリップ同士の
  共通点・パターン・関係性に気づく
  「自分、こういうの好きだったんだ」
  ↓
⑤ 記憶の蘇り
  あのとき、どこで、なぜ気になったのか
  クリップが記憶のトリガーになる
```

---

### 2.2 体験の3本柱

#### 体験① 記憶の蓄積
写真全体ではなく「気になった部分だけ」を切り取るから、記憶の純度が高い。ページをめくるたびに、あのとき・あの場所・あの感覚が戻ってくる。写真日記とは異なり、全体を記録しようとしなくていい。断片だからこそ、記憶の純度が高い。

#### 体験② 自分のコレクション
「赤いもの」「丸いロゴ」「和な感じのパッケージ」など、自分でテーマを決めて収集できる。集まっていくほどに達成感があり、ブックが育っていく感覚がある。コレクターとしての楽しさと、自分の感性の可視化が同時に起きる。

#### 体験③ 偶発的な発見
能動的に「記録しよう」とせず、気になったから貼っただけなのに、後から意味が見えてくる。積み重なったクリップ同士に関係性・共通点・パターンが浮かび上がる。その偶発性こそがScrappAの醍醐味。

---

### 2.3 断片性という独自価値

Scrappaが最もユニークな点は**「断片を切り取る」**という行為が設計の中心にあること。

- 写真日記 → 全体を記録する
- カメラロール → 全体が保存される
- **Scrappa → 気になった部分だけを切り取る**

断片だからこそ、記憶の純度が高く、見返したときの引っかかりが強い。

## 4. 競合との差別化

### 4.1 ポジショニングマップ

```
                    他人のコンテンツが主役
                           │
              Pinterest     │    Cosmos
              Instagram保存 │    （クリエイター向け）
                           │
自分で作らない ─────────────┼───────────────── 自分で作る
（保存・ブックマーク）      │          （デコ・レイアウト）
                           │
                           │  Picrap    GoodNotes
                           │  （思い出アルバム）
                    自分のコンテンツが主役
                           │
                       Scrappa ●
                    （日常の断片・収集）
```

Scrappaは「自分のコンテンツが主役」×「作ること自体はシンプル」というポジションで、どの競合とも重なりません。

---

### 4.2 主要競合との比較

#### Pinterest / Instagram保存

| 観点 | Pinterest / Instagram | Scrappa |
|---|---|---|
| コンテンツの主役 | 他人の画像 | 自分で撮った断片 |
| 目的 | インスピレーション収集・発信 | 記憶の蓄積・コレクション |
| ノイズ | 広告・AI画像・おすすめが多い | 自分の断片だけ |
| 見返す体験 | アルゴリズムが介入する | 自分が貼った順に積み重なる |

**一言：** PinterestとInstagramは「他人のコンテンツを見る場所」。Scrappaは「自分の断片を残す場所」。

---

#### Cosmos

2023年に立ち上がり、2025年にはApp Storeのデザインカテゴリで28カ国1位を獲得した最注目の競合。

| 観点 | Cosmos | Scrappa |
|---|---|---|
| 対象ユーザー | クリエイター・デザイナー | 感度より「気になった」が基準の一般人 |
| コンテンツ源 | Web・SNS・他プラットフォームから収集 | 自分で撮影した写真が主役 |
| 目的 | インスピレーション発見・ムードボード | 日常の記憶の蓄積・コレクション |
| UI | ダークでクールなデザイナー向け | アイボリー・方眼・ノートの温かみ |
| 日本展開 | 海外メイン | 日本語・日本文化に寄せた設計 |
| 交換機能 | なし | カードを物理的に交換できる |

**一言：** Cosmosは「世界中のビジュアルからインスピレーションを見つける場所」。Scrappaは「自分が日常で切り取った断片を育てる場所」。

---

#### Picrap（日本製スクラップブックアプリ）

| 観点 | Picrap | Scrappa |
|---|---|---|
| 作業量 | レイアウト・デコを自分で作り込む | 撮って貼るだけ・シンプル |
| 目的 | 思い出を「完成品」として作る | 断片を積み上げていく過程が楽しい |
| ソーシャル | なし | フレンド・いいね・カード交換 |
| 継続性 | 作り込むのが大変で続かない可能性 | 低い敷居・続けやすい |

**一言：** Picrapは「完成させるアルバム」。Scrappaは「育てていくブック」。

---

### 4.3 Scrappaだけの独自優位性

競合と比較したとき、Scrappaにしかない要素は4つ。

**① 自分で撮った断片が主役**
他のどのサービスも「自分が撮影した写真の断片」を主役に置いていない。

**② 正方形トリミングによる統一感**
切り取って貼るという行為がアプリの設計に組み込まれている。断片を切り取ることで記憶の純度が上がる。

**③ ノートという器の体験**
ページをめくる・積み重なるという物理的なスクラップブックの感覚をデジタルで再現している。

**④ カード交換というコレクター文化**
他のサービスにない、物理的な所有権移転を伴う交換機能。コレクター文化と相性が良い。

---

### 4.4 差別化の一言まとめ

> 「自分が切り取った日常の断片だけで作る、育てるスクラップブック。」
>
> Pinterestは他人の画像を集める場所。Cosmosはクリエイター向けのムードボード。Scrappaは自分が日常で出会ったものの断片を、自分だけのブックに貼り続ける場所。

## 5. PR戦略

### 5.1 全体方針

ゼロ予算・個人開発・MVP完成後スタートという条件での最適戦略は**「作る過程と使う体験を公開すること」**。広告ゼロでも、コンセプトとビジュアルが強ければ共感で広がる。ScrappAはUIがそのままコンテンツになる強みがあるため、見せるだけで伝わるアプリ。

**プラットフォーム**
- **Instagram（Scrappa専用アカウント）**：世界観・ビジュアルで伝える
- **X（個人アカウント）**：個人開発クラスタへのリーチ・テキストで共感を狙う

---

### 5.2 フェーズ別PR計画

#### Phase 0：MVP完成直前（今すぐ）

完成を待たずにアカウントだけ作っておく。公開時に0フォロワーからスタートするのと100フォロワーからスタートするのでは初速が全く違う。

- [ ] Instagram に Scrappa 専用アカウントを開設
- [ ] プロフィールに「切り取って、貼って、私になる。」を設定
- [ ] ロゴ・アイコンを設定

---

#### Phase 1：MVP公開直後（最初の100人を集める）

**目標：使ってくれる最初のユーザーを50〜100人集める**

**Instagramでの発信**

ビジュアルで世界観を伝えることを最優先にする。

| 投稿タイプ | 内容 |
|---|---|
| スクリーンショット投稿 | リングノートのUIをそのまま見せる。「このアプリおしゃれ」という第一印象を狙う |
| カルーセル投稿 | 街で撮ったパッケージ → トリミング → ブックに並んだ状態の一連の流れ |
| リール動画 | アプリの使い方を30秒で見せる（詳細は第6章） |

狙うハッシュタグ：
`#手帳好きと繋がりたい` `#パッケージ好き` `#デザイン好き` `#コレクション` `#スクラップブック`

**X（個人アカウント）での発信**

個人開発クラスタへのリーチを狙う。

投稿パターンA：開発リリース告知

#個人開発 #webapp
```

投稿パターンB：コンセプトへの共感を狙う
```
カメラロールに眠ってる写真、ありませんか。
「これいいな」と思って撮ったけど
誰にも見せることなく埋もれていく写真。
Scrappaはそういう写真のための場所です。
```

投稿パターンC：使い方を見せる
```
Scrappaの使い方はシンプルです。
①気になるものを撮る
②気になる部分だけトリミング
③ブックに貼る
それだけ。整理しなくていい。続けるほど自分が見えてくる。
```

狙うハッシュタグ：
`#個人開発` `#webapp` `#デザイン好き` `#パッケージデザイン` `#手帳好き`

---

#### Phase 2：口コミを生む仕掛け（公開1〜3ヶ月後）

**目標：ユーザーが自然に発信したくなる仕掛けを作る**

**施策①：シェアしたくなるUIの活用**

ブックのスクリーンショットがそのままSNS映えするUIを設計済み。「Scrappaで作りました」という投稿が自然に広がることを期待できる。

```
ユーザーが自分のブックをInstagramにシェア
  ↓
「このアプリなに？」と興味を持った人がアプリを試す
  ↓
自分のブックをシェア → 繰り返し
```

**施策②：コレクターコミュニティへのアプローチ**

パッケージコレクター・シールコレクター・手帳クラスタはすでにXやInstagramで活発に活動している。このコミュニティに自然に入り込む。

- パッケージ収集タグ（`#パッケージ収集` `#お菓子パッケージ`）で投稿しているユーザーにいいね・リプライ
- 「こういう使い方できますよ」と押しつけがましくなく伝える

**施策③：noteでコンセプト記事を書く**

検索流入を狙った記事を1本書くだけで長期的な効果がある。

おすすめタイトル案：
- 「カメラロールに眠った写真を『自分の本』にする話」
- 「デジタルスクラップブックを作った理由」
- 「切り取って貼るだけ。整理しない収集術」

---

#### Phase 3：継続的な成長（3ヶ月以降）

**目標：月間アクティブユーザーを着実に増やす**

**施策①：定期的なアップデート告知**

新機能（ドラッグ並び替え・交換機能など）のリリースごとにX・Instagramで告知。個人開発は「作り続けている」こと自体がコンテンツになる。

**施策②：ユーザーのブックを紹介する**

許可を得てユーザーのブックをScrappa公式アカウントで紹介する。ユーザーにとって嬉しい体験になり、口コミが広がる。

**施策③：ProductHuntへの投稿**

英語圏への露出も視野に入れるなら、ProductHuntへの投稿は無料で大きなリーチを狙える。日本発のアプリとしての独自性が武器になる。

## 6. リール動画構成

### 6.1 基本情報

| 項目 | 内容 |
|---|---|
| 尺 | 30秒以内 |
| スタイル | ナレーションあり（声あり） |
| 音楽 | Lo-fi / acoustic / indie pop系（温かみのある雰囲気） |
| タイトル | 「切り取って、貼って、私になる。」 |

---

### 6.2 シーン構成

#### [0〜3秒] フック（つかみ）
```
映像：カメラロールの写真が大量にある画面
ナレーション・テロップ：「撮ったのに、埋もれてる写真ありませんか」
```
冒頭3秒で離脱を防ぐ。「あるある」の共感から入る。

---

#### [3〜8秒] 問題提起
```
映像：街を歩いているシーン → 気になるパッケージや看板を発見
ナレーション・テロップ：「気になったものを、撮る」
```
ユーザーの日常行動と重ねる。

---

#### [8〜15秒] 体験の核心（最も大事な場面）
```
映像①：アプリを開く → ＋ボタンをタップ
ナレーション：「気になる部分だけ、切り取って」

映像②：正方形にトリミングする操作（アプリの画面収録）

映像③：ブックに貼られていくクリップ
ナレーション：「リングノートに、貼るだけ」
```
アプリの操作を実際に見せる。難しくなさそうと思わせることが重要。

---

#### [15〜22秒] 積み重なりの魅力
```
映像：ブックのページをめくるシーン（クリップがびっしり並んでいる状態）
ナレーション・テロップ：「貼り続けると、自分が見えてくる」
```
ページをめくる演出がそのまま映像コンテンツになる。

---

#### [22〜27秒] フレンド機能の紹介
```
映像：他ユーザーのブックを閲覧・いいねを押すシーン
ナレーション・テロップ：「フレンドのブックも、のぞける」
```
ソーシャル要素を一瞬だけ見せる。

---

#### [27〜30秒] CTA（行動喚起）
```
映像：アプリのロゴ・ブックの全体画面
テロップ：「Scrappa　無料で始める」
         「プロフィールのリンクから↓」
```

---

### 6.3 撮影素材リスト

| 素材 | 内容 | 撮影方法 |
|---|---|---|
| 街歩き映像 | パッケージ・看板・雑貨などを発見するシーン | スマホで撮影 |
| アプリ操作画面 | アップロード・トリミング・ブック表示 | 画面収録（iOS/Android） |
| ブックめくり | クリップが並んだページをめくるシーン | 画面収録 |
| フレンド画面 | 他ユーザーブック閲覧・いいね操作 | 画面収録（ダミーデータでOK） |

---

### 6.4 音楽の方向性

- テンポ：ゆったり〜中程度（速すぎない）
- ジャンル：Lo-fi・acoustic・indie pop
- 雰囲気：温かみがある・懐かしい感じ
- 調達方法：Instagram Reelsの楽曲ライブラリから「lo-fi」「acoustic」で検索

---

### 6.5 制作ステップ

```
Step 1：アプリの画面収録を先に撮る
         （ダミーデータでいいので実際の操作を収録）

Step 2：街歩き映像を撮る
         （パッケージ・雑貨・看板などを意識して撮影）

Step 3：CapCut / iMovie などで編集
         テロップ → ナレーション → 音楽の順で乗せる

Step 4：Instagramに投稿
         キャプションにキャッチコピーとリンクを記載
```

---

### 6.6 投稿キャプション案

```
切り取って、貼って、私になる。
